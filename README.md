# Face De-Identification and rPPG Signal Integrity Research

This repository contains the research code and datasets developed during the face de-identification (De-ID) project in collaboration with **A*STAR**. The primary objective was to evaluate how various face-swapping and diffusion-based de-identification methods impact the extraction and accuracy of remote Photoplethysmography (rPPG) signals.

---

## Repository Structure

### 1. `AI_gen_faces/`
Contains the target "source" faces used for swapping. These are high-quality, AI-generated identities used to replace original subjects in the datasets to ensure privacy.

### 2. `baselining/`
Includes scripts used to establish a performance benchmark.
* **Methodology:** Runs standard rPPG translation via the **rPPG-Toolbox** on original, unmodified datasets to provide a ground-truth performance metric for comparison.

### 3. `facefusion/`
Houses higher-resolution and hybrid De-ID experiments.
* **`MASS_RUN.py`**: Automated script for performing **512x512 pixel** face swaps across the dataset.
* **`MENTOR_SUGGESTED_METHOD.py`**: Implements a hybrid approach using **face-diffusion** on top of face-swapping to analyze if diffusion further obscures identity or significantly degrades rPPG signal quality.

### 4. `rPPG-Toolbox/`
The core evaluation framework, modified to include specific De-ID and batch-processing scripts.
* **`128x128_Face_swapper.py`**: Performs face swapping at 128x128 resolution using the `inswapper_128.onnx` model.
* **`MASS_DE-ID_SWAPPER.py`**: A batch processing script designed to face-swap entire dataset folders efficiently for large-scale testing.
* **`config/`**: Contains all `.yaml` configuration files. These files define the parameters for testing swapped-face videos against ground truth data.

### 5. requirements.txt
This file contains all the modules' prerequisites and versions to run on a virtual environment.

## Evaluation Metrics

The experiments evaluate De-ID effectiveness and signal retention based on the following metrics generated by the toolbox:

| Metric | Description |
| :--- | :--- |
| **MAE** | Mean Absolute Error |
| **RMSE** | Root Mean Square Error |
| **Pearson** | Pearson Correlation Coefficient ($r$) |
| **SNR** | Signal-to-Noise Ratio |

---

## Workflow

1.  **Baseline Generation:** Run scripts in `/baselining` to obtain initial rPPG results from the raw data.
2.  **De-Identification:** * Use `rPPG-Toolbox/MASS_DE-ID_SWAPPER.py` for standard 128x128 swaps.
    * Use `facefusion/MASS_RUN.py` for high-res 512x512 swaps.
    * Use `MENTOR_SUGGESTED_METHOD.py` for the diffusion-enhanced approach.
3.  **Evaluation:** Update the `.yaml` files in `rPPG-Toolbox/config/` to point to the processed video directories and run the evaluation suite to generate performance metrics.
